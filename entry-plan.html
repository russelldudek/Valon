90-day entry plan

Earn the right to standardize by shipping one complete engagement.

DAYS 1–15

Enter and map

  • Embed with the signed client and shadow operators, executives, data owners, compliance, and technical stakeholders.
  • Map workflow, exceptions, informal authority, data gaps, failure cost, and change anxiety.
  • Write one shared engagement contract: outcome, owner, user, authority, evidence, recovery.

Outputs: Live-operation map; authority map; baseline; bounded first use case.

DAYS 16–30

Build the first real workflow

  • Choose the smallest use case with real value and platform learning.
  • Build directly with orchestration, APIs, retrieval, deterministic rules, and human review.
  • Define evaluation, auditability, fallback, escalation, and owner.

Outputs: Working prototype; evaluation set; authority boundary; recovery design.

DAYS 31–60

Operate to proof

  • Run with real operators and real exceptions.
  • Track quality, cycle time, rework, trust, exception burden, economics, and recovery.
  • Treat operator behavior as workflow-fit evidence; iterate until the result exists.

Outputs: Outcome scorecard; exception taxonomy; adoption record; hardened workflow.

DAYS 61–90

Return and repeat

  • Separate client-specific context from reusable capability.
  • Package workflow logic, authority, evaluation, recovery, and adoption learning.
  • Install engagement review cadence and future strategist quality criteria.

Outputs: Platform-return package; playbook; next-use-case pipeline; hiring standard.
